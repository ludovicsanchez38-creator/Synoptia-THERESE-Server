## Problèmes critiques (action immédiate)

1. **Fuite cross-tenant data.py** - select(Message) sans filtre user_id/org_id
2. **Seed admin/admin** - credentials par défaut prévisibles
3. **6 endpoints RGPD sans auth** - anonymize/export/consent accessibles sans JWT
4. **Image Docker 13GB** - swap saturé, risque OOM

## Plan d'action

### Semaine 1 (Quick Wins)
- Corriger filtrage cross-tenant data.py
- Supprimer seed admin/admin
- Ajouter auth sur 6 endpoints RGPD
- Brancher prompt_security sur routers LLM
- Headers CSP dans Caddy
- Compression gzip/brotli Caddy

### Semaine 2-3 (Structurel)
- Pipeline CI/CD GitHub Actions
- Dockerfile multi-stage (<500MB)
- Remplacer except Exception (216→<10)
- Migrer datetime.utcnow() (175 occurrences)
- Tests frontend Vitest (20+ tests)
- Pagination tasks/contacts

### Mois 2+ (Évolutions)
- Backups externalisées S3
- Circuit breaker + retry LLM
- Markdown dans le chat
- Mentions légales + politique confidentialité
- Recherche globale Ctrl+K

## Résumé exécutif

Thérèse Server est un MVP fonctionnel avec une architecture monolithique modulaire solide (71/100) et une sécurité de base correcte (JWT, bcrypt, RBAC, sandbox), mais trois failles critiques nécessitent une correction immédiate : fuite cross-tenant, seed admin par défaut et endpoints RGPD non protégés. Le point faible majeur est le DevOps (35/100) avec zéro CI/CD, une image Docker de 13GB et aucune sauvegarde externalisée. Le projet obtient **59.7/100** ; atteindre 75/100 est réaliste en 4-6 semaines.
